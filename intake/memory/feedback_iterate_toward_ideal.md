---
name: Iterate toward ideal, not first draft
description: Never settle for first implementation take — always strive for the higher idealized formation before building
type: feedback
---

First is not best. The user's original instinct, or Claude's first/second/third take, are not the goal — the higher idealized formation of ALL is.

**Why:** Rushing to implement the first workable design produces architecture that needs reworking later. The user values getting the structural placement and conceptual integrity right before any code is written.

**How to apply:** When designing systems, don't treat the first viable architecture as the plan. Challenge organ placement, naming, boundaries, and conceptual fit. Present options rather than defaulting to the obvious location. Resolve foundational questions (where does this live? what is this organ's identity?) before file structures and implementation steps.
