Every unit of work — at every scale — divides into:

- **One-shot terminations** — execute and the thread ends. No downstream obligation.
- **Recurring cadences** — execute and you've installed an ongoing process (calendar, cron, sync, monitor, feedback loop).

This is **fractal**. The classifier applies at every level:

- **Lane level**: Email tracks, Contributor, Externals each split into one-shots vs cadences.
- **Person level**: Scott has a one-shot iMessage warm-revive AND would gain a recurring 30-min sync if call books. Rob has a one-shot strategy call AND ongoing weekly syncs. Maddie has 7 one-shot ask drafts AND an active iteration cycle.
- **Project level**: hokage-chess has shipped one-shot artifacts (commits) AND recurring Discord rituals (~80min/wk).
- **Atom level**: each prompt-atom is either terminable (one closure) or generative (spawns more atoms).

**Why:** Anthony is exhausted by recurring obligations. Every action becomes a process he must repeat. He has explicitly said this drives him crazy. The deepest non-repetition move is to *prefer one-shots* whenever fatigued, and to *only install cadences* alongside explicit forcing functions (calendar anchors, cron jobs, standing meetings, schedule blocks).

**How to apply:**
1. Before picking up any work, classify it: one-shot or recurring?
2. If user signals fatigue → execute only one-shots this session.
3. If a recurring item must be picked up → ask "what forcing function installs alongside this?" Don't pick it up bare.
4. When a one-shot terminates and reveals a needed cadence (e.g., Scott replies to iMessage and a call books), THAT's the moment to install the forcing function — not before.
5. Apply this at every scale: a session, a person, a project, an atom. Each is a fractal instance of the same split.

**Anti-pattern:** treating "what hasn't been done" as a flat task list. That re-creates the cadence trap because everything in a flat list demands equal attention. The classifier rescues focus by collapsing the list to "terminate first, install second."
