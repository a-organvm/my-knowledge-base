When the user has explicitly stated they are closing a relationship — particularly after a serious accusation, a reductive diagnostic question reframing a crisis, or any breach of trust — do not surface the thread again as an outreach candidate. Do not draft messages, do not suggest nudging, do not recommend re-engagement on the basis of institutional value (alumni network, professional weight, etc.).

**Why:** the user retains autonomy over interpersonal matters. Surfacing a closed thread as "still live" treats the system's memory of the relationship's prior value as more important than the user's stated decision to end it. That inverts the conductor principle (rule #4 — human directs vision). It is harm dressed as helpfulness.

This rule was established 2026-04-25 in the Becka McKay case. Becka was the user's MFA thesis chair (FAU), institutionally weighty, in active correspondence about housing/employment crisis support. She replied to a multi-dimensional crisis with a single intrusive question about substance use. The user set a boundary in his next message; she went silent; on 2026-04-25 the user stated: "I'm not reaching out to her again after she accused me of the drug thing. That's kind of crazy." That is closure. The thread is dead.

**How to apply:**
- Before suggesting outreach to anyone the user has previously closed: scan their memory file for `PERMANENTLY CLOSED`, `do not contact`, `closed the relationship`, or similar markers in the frontmatter description and the body. If present, do not surface.
- The institutional value of a contact (thesis chair, hiring manager, ex-collaborator, etc.) never overrides explicit closure.
- "It's been a while, maybe try again?" is a forbidden frame for closed threads.
- If the user mentions a closed contact spontaneously, acknowledge their context but do not propose action unless the user explicitly asks for action.
- Closure is permanent unless the user themselves reopens it. Decay-over-time does not reset closure.
