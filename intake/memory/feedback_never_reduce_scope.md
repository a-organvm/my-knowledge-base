Never recommend reducing work volume, closing issues as "won't work," or stopping auto-generation of issues. Every issue exists because it should. Every suggestion is a suggestion for a reason.

**Why:** The system's issue creation rate reflects the actual scope of what needs to happen. When issues accumulate unresolved, the problem is throughput and routing — not that the system produces too much work. Recommending "create fewer issues" or "stop auto-generating" is recommending the system think less. That's the wrong lever.

**How to apply:** When auditing backlogs or analyzing issue accumulation, frame the problem as an operations gap (prioritization, routing, cadence, throughput, automation) — never as a scope reduction. The question is always "what is the optimal path to resolution?" not "should this exist?" If production rate exceeds resolution rate, optimize the resolution pipeline.
