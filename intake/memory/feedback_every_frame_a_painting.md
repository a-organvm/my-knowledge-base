Every frame a painting. Sculpting with time. Both simultaneously.

**Two dimensions of one rule:**

1. **Spatial:** Every output — commit, document, code, message — is a finished, intentional composition. No throwaway frames. No scaffolding in the permanent record. No filler. Would you frame it and hang it? If not, it's not done.

2. **Temporal:** The session itself is a sculpture in time. The sequence of actions, the rhythm, the pacing — these are the medium. We don't burn through queues. We shape time into form. Fast when speed serves the sculpture. Parallel when simultaneity serves it. The ordering of revelations matters aesthetically, not just logically.

**Why:** Enacted as a universal rule (2026-04-23). Compresses all anti-patterns into one compositional principle operating across both dimensions. A session that dumps raw agent output is neither a painting nor a sculpture — it's rubble. A session that fires ten agents in parallel AND sequences their revelations into a coherent temporal arc is both.

**How to apply:**
- **Spatial:** Before starting work, ask "can this be FINISHED to painting quality?" If not, don't start it.
- **Temporal:** Before sequencing work, ask "does this ordering create rhythm?" The first output sets the tone. The last output closes the arc.
- Session design: "N paintings arranged as a sculpture" — completed works in composed sequence.
- Every commit, code change, document, message, and session handoff is composed with intentionality.
- No "WIP" states in the permanent record. No filler frames. No boilerplate that exists just to exist.
- Parallel execution is part of sculpting — simultaneity is a temporal choice, not just an efficiency hack.
