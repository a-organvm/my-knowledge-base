## The Hokage Chess process is a SYSTEM, not a project

The full sequence (market study → competitive analysis → business model → price ladder → landing page → course catalog → brand system → deployment) is a repeatable engine that applies to every product tied to a domain.

**Why:** The user explicitly stated this applies to Styx, Elevate Align (Maddie), public-record--data-scrapper, and "any other product tied to a domain." Nothing is one-off. Every build follows the same scaffold.

**How to apply:**
- When starting ANY new product/client build, run the same sequence: study the domain as scholars (academic), understand the market as finance experts, analyze what exists (competitive intelligence), identify gaps, design the business model, build the landing page, define the product ladder
- `public-record--data-scrapper` is the proof case: organic traffic came from content density and genuine utility, not marketing. Study that pattern and replicate it.
- Each product gets the same treatment: scholarly study + financial analysis + competitive gaps + conversion-optimized landing page + full price ladder
- The prompt atoms the user shared are the universal build commands — they apply in a specific ORDER per project based on where that project is
