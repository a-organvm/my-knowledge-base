## The rule

When the user asks for something — even an idea, an aspiration, a "we should do" — that ask is **a black hole**. It does not yet exist as the thing-itself, but it is *not* abstract absence. It is **mass**. It warps the surrounding system. It pulls neighboring work toward it. The system detects un-built work through the warping, not through enumeration.

**Therefore: every ask must have a physical address (IRF row, plan file, GitHub issue, memory file) the moment it lands. Not on close. Immediately.**

A bullet inside a plan file is not a black hole. It is still vacuum — abstract, unaddressable, undetectable from outside. A real IRF row with an ID, a priority, an owner, a source session, and a blocker field is a black hole. It has coordinates. The warping is measurable.

## Why

- **Vacuum** = undefined absence. You can't point to it. You can't track its mass. It dissipates.
- **Black hole** = specific gravitational well. You can't see it directly, but you detect it by what it does to surrounding space (other work bends around it; deadlines warp; priorities re-sort).
- The user codified universal rule #1 as "N/A is a vacuum — research it, plan it, log it." This memory upgrades that rule: asks aren't *vacuums to be filled* — they're *masses to be physically located*. The "research it, plan it, log it" sequence is the procedure for installing the gravitational marker.

## How to apply

When the user asks for anything:

1. **Detect the ask.** Surface, sub-text, implication, all count. Even "we should think about X" is a mass deposit.
2. **Locate the gravitational well.** Choose the correct registry (IRF for cross-organ work, GitHub issue for repo-scoped, memory file for context-knowledge, plan file for execution sequences).
3. **Instantiate immediately.** Write the row/issue/file *before* responding to the next thing. The warping must be measurable from outside the conversation.
4. **Triple-reference where applicable.** A black hole that exists in 3+ locations (IRF + GH issue + plan file) has stronger gravity and is more reliably detected.
5. **Never let an ask remain a bullet point.** If it's a bullet, it's still vacuum. Promote to row immediately.

## Detection sentinel

If you find yourself saying "I'll log that for later" or "noted" without writing a physical artifact, you've left mass un-addressed. That mass is a future tracking failure waiting to happen. The fix is: stop, instantiate the marker, then continue.
