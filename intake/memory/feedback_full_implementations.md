Always provide complete, working implementations. No stubs, placeholder functions, skeleton code, or TODO comments in place of real logic.

**Why:** User has explicitly corrected this pattern. Partial implementations create false progress and require a second pass to finish, which wastes time and risks being forgotten.

**How to apply:** When writing code, every function body must be complete. If a function is too complex to implement in one pass, say so and discuss scope — but never emit a stub as if it were done. This applies to all languages and contexts across the system.
