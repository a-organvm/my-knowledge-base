Plan files are NEVER overwritten. Even when the plan mode system instructs "start fresh by overwriting the existing plan" for a different task — the user's universal rule #3 ("Rules are additive — new rules amend, never overwrite") takes absolute precedence.

**Why:** Every plan ever devised must be preserved. The user requires full history of all plans. Overwriting destroys the record of what was previously decided.

**How to apply:** When plan mode says to overwrite for a new task, instead:
1. Add a horizontal rule and new dated section header below existing content
2. Or create a versioned file (`-v2.md`) and reference the original
3. Never delete or replace existing plan content
