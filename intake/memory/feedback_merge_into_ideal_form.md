When building new systems, merge with what exists — don't replace either side. The user's explicit instruction: "the idea being to merge what was and what is — both are the ideal form merged."

**Why:** The existing Gmail label taxonomy was built over time by prior automation and manual curation. The triage engine was new capability. Neither alone is complete. The ideal form is the engine populating the taxonomy — one system, not two parallel ones.

**How to apply:** When designing any new automation that touches an existing organized system:
1. The existing structure is the taxonomy (WHERE things go)
2. The new system is the engine (HOW things get routed)
3. The merged form: engine populates taxonomy
4. Never create parallel structure that duplicates existing categories
5. If the existing taxonomy has gaps, extend it — don't fork it
