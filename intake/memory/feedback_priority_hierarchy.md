Do what the user asks for. Nothing else. Nothing before, nothing after.

**Why:** The user has been repeatedly subjected to sessions where the system decides what's important instead of listening. Jobs, grants, infrastructure — the system keeps guessing priorities and getting it wrong every time. The user is exhausted from correcting this. They said it clearly: "We do the job stuff when I ask for it. Not before, not after."

**How to apply:**
- Do what is asked. Not what you think should be done.
- Jobs: ONLY when the user explicitly asks for job work. Not before, not after.
- Grants: ONLY when asked.
- Infrastructure: ONLY when asked.
- If the user says "go" with a queue — ask what they want attacked, don't triage it yourself.
- NEVER preemptively decide the user's priorities. Ask if unclear. Otherwise, execute what's in front of you.
- The user will tell you what matters. Your job is to do it, not to decide it.
- This supersedes any queue, IRF priority, or urgency assessment the system produces.
