## VLTIMA MATERIA is a platform, not a personal tool

Crystallized 2026-04-24 after a session that started with a broken skills symlink and ended with a fundamental reframing.

**The core insight:** VLTIMA MATERIA is not Anthony's studio system. It's a meta-system that becomes whatever the person wielding it needs: a multimedia studio, a law practice, a research lab, a farm, a consultancy. The platform has no fixed shape — it takes the shape of whoever holds it. ORGANVM is one deployment (the user's instance). The platform underneath is generic, nameless, for anyone.

**Why:** "For someone else, it's something entirely different. That's the main fucking crux of this whole thing."

**The four laws (ultima materia physics):**
1. Identity is function (produces/consumes), not name
2. Location is ephemeral — discovered, never stored as truth
3. Composition is declaration — edges from identity contracts, not config
4. Everything enters stripped — reduced to identity + edges, all else discarded

**How to apply:** Every architectural decision must pass: "does this work for someone who isn't Anthony, whose system isn't called ORGANVM, whose repos have different names?" If no, it's too specific. The platform primitives (resolver, topology, seed contracts) must be generic. System-specific branding belongs in the deployment layer, not the platform.

**Revenue implication:** The product is the platform itself — a personal AI operating system anyone can deploy. Client work (Maddie, consulting) runs through one instance but isn't the product. The platform IS the product.

**Current state:** Resolver + topology module built and working (108 repos, 174 capabilities). Still branded "organvm" — needs renaming to be generic. Bootstrap resolver is brand-agnostic (plain shell, no system name in logic).
