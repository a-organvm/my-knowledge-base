## The Tenet Protocol

When a force is set off in one direction, the opposite, opposing direction gets set off as well. Named after Christopher Nolan's Tenet — time flowing forward and backward simultaneously.

**In practice:**
- Outbound client acquisition → Inbound portfolio/booking system that ATTRACTS clients
- Mechanical code dispatch (Codex builds) → Governance design that VALIDATES the builds
- Research going out (Gemini scavenges) → Ingestion pipeline that PROCESSES what returns
- Application submission → Relationship building with the people behind the application
- System expansion → System compression (documentation, cleanup, archiving)
- Job seeking (external dependency) → Client building (self-sovereignty)

**How to apply:** Every dispatch table entry should have a TENET column. When dispatching Task A to Agent X, simultaneously dispatch Task A' (the opposing force) to Agent Y. The pair completes the loop.
