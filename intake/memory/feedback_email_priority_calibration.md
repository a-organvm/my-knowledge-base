User hasn't spotted a single email that seems "that important" in a batch of 139+ that included payment failures, collection notices, API key exposures, and student loan default warnings.

**Why:** The user's threshold for "important" is much higher than the system's default triage. Financial notices and security alerts that feel urgent to a system are routine noise to someone in survival-mode crisis. Over-escalating drowns out the signal.

**How to apply:** When triaging email, calibrate to the user's actual priority scale. Most financial/security notices are MEDIUM at best, not HIGH. TRUE high priority = a real human reaching out, a time-sensitive opportunity, or something that will cause irreversible damage if ignored THIS HOUR. Everything else is This-Week or Someday. Don't cry wolf. (2026-04-23)
