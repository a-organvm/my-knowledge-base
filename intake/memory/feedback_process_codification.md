Everything we do is meant to be processed as process and codified as such. We don't do anything once.

**Why:** If an action is worth doing, it will need to happen again. Doing it manually the first time without codifying the process means the second time requires re-explaining, re-discovering, re-implementing. This is the operational gap the user keeps hitting — rules stated, not enforced; fixes applied, not codified; solutions built, not integrated.

**How to apply:**
1. When fixing something (e.g., installing a missing dependency), also fix the SETUP so it's never missing again
2. When enforcing a rule (e.g., 24-hour freshness), codify it in CODE, not just in memory or documentation
3. When building a workflow (e.g., job sourcing), make it a single repeatable command with sensible defaults
4. When capturing knowledge (e.g., fish boil atoms), route it into the system that governs it, not a notes file
5. Before declaring a task done: "Is this a process now, or was it a one-shot?" If one-shot, it's not done.

This is Universal Rule #7 restated at the operational level: "Everything is a loop — no action is single-shot."
