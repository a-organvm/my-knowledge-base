## Dispute Resolution Flow

```mermaid
sequenceDiagram
    participant User
    participant API as NestJS API
    participant Stripe
    participant Judge as The Judge (Desktop)

    User->>API: POST /contracts/:id/dispute
    API->>Stripe: holdStake($5 appeal fee)
    API->>API: Create dispute record
    API-->>User: { appealStatus: FEE_AUTHORIZED }

    Judge->>API: GET /admin/disputes
    API-->>Judge: Dispute queue with metadata

    Judge->>API: GET /admin/disputes/:id
    API-->>Judge: Detail + signed media URL + Fury votes

    Judge->>API: POST /admin/disputes/:id/resolve
    Note over API: outcome: UPHELD | OVERTURNED | ESCALATED

    alt UPHELD
        API->>Stripe: captureStake (fee revenue)
    else OVERTURNED
        API->>Stripe: cancelHold (refund fee)
        API->>API: Penalize incorrect Furies (-10 IS)
    else ESCALATED
        Note over API: Hold maintained, status updated
    end
```
