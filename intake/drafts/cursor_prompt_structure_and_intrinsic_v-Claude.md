## What “completeness” can mean (pick one per audit)

- **Exhaustive coverage of a declared universe**  
  Example universe: `Workspace/organvm/**/docs/**` only, excluding `.archive/`. If the universe is underspecified, “100% done” is meaningless.

- **Closed-world completeness** (strong)  
  “There is no additional writing outside this set” *within the defined universe*, proven by a deterministic scan + invariants + negative tests.

- **Open-world “best effort”** (weak)  
  A large glob count. Useful for triage, not proof.

If you do not state the universe and exclusions up front, any completeness claim is non-falsifiable.

## What counts as *proof* for someone else’s audit

A credible audit would publish:

1) **A canonical definition** of “writing”  
   Extensions (`.md`, `.tex`, certain `.txt`?), and whether generated/vendor files count.

2) **A declared scope and exclusions**  
   Explicitly list ignored roots (`.git`, `node_modules`, `vendor/`, `dist/`, `build/`, `.archive/`, `intake/`, etc.), and why.

3) **A deterministic method** that others can re-run and get the same file list and hashes  
   - file listing + per-file `SHA-256` (or a Merkle root over sorted paths)  
   - a manifest with counts by category

4) **Invariants the corpus must satisfy** (examples)  
   - no duplicate canonical paths (same “logical document” in two places)  
   - no empty required sections in governance docs (if you define that)  
   - every SOP has an owning repo + supersedes link (if that’s your law)

5) **Negative / adversarial tests** (see below)  
   Not just “I found 10,000 files,” but “I attempted 10 common hiding tricks and found none.”

6) **Provenance and freshness**  
   When was the scan run, on which machine path, and under what tool versions.

Without (2)–(3), a completeness claim is mostly vibes.

## How we (or a reviewer) can be tricked: the threat model

### 1) **Universe smuggling (the biggest one)**  
A report looks exhaustive because it hits “tons of `.md`,” but it never searched `~/Documents`, an external drive, a private org repo, a cloud doc store, or a different branch.  
*Symptom:* huge counts + warm feelings; *still not closed-world*.

### 2) **Name/path hiding**  
Writing stored as:
- `README` with no extension  
- `*.mdx` / `*.org` / `*.typ` / `*.ipynb` / PDF-only  
- content embedded in `*.html`, `*.json`, or SQL dumps  
*Symptom:* glob undercounts.

### 3) **Content hiding inside non-text artifacts**  
PDF/DOCX, images, slide decks, email exports, Notion zips, chat exports.

### 4) **Shadow duplicates masquerading as “done”**  
The same “canonical” text exists in 3 places (repo + archive + distribution copy). A naive inventory inflates “done,” while a canonical one might still be missing a single authoritative locus.  
*Symptom:* high file counts + unclear authority.

### 5) **Tool truncation / search limits (silent)**  
If any step caps results or truncates output, you can *believe* you saw the full set while missing tail files.

### 6) **“Implementation theater”**  
You see pipelines and contracts and assume runtime guarantees exist, but some paths are scaffolds/TODO, or the evaluation gates are not actually enforced in CI.  
*Symptom:* impressive architecture docs, weak enforcement.

### 7) **Hand-wavy “audit complete” without artifacts**  
A narrative audit (“all repairs complete…”) is not a proof unless backed by a manifest, tests, and reproducible diffs.

### 8) **Symlinks, ignored dirs, and permission silos**  
A scan that skips permission errors will silently undercount.

## How to check “proof of completeness” when someone else does the work (review checklist)

Ask for these deliverables, and treat absence as failure:

- **Manifest** with: path, size, mtime, hash, category, canonical flag, supersession pointer  
- **Scope statement** in one paragraph (what is in/out)

- **Delta report** if rerunning later: new/moved/deleted/renamed  
- **Category reconciliation**: governance vs theory vs spec vs operations — with explicit cross-links  
- **Negative-search evidence** for the top hiding tricks in your environment (at least attempted)
