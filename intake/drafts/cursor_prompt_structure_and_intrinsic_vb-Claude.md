## What counts as _proof_ for someone else’s audit

A credible audit would publish:

1) **A canonical definition** of “writing”

2) **A declared scope and exclusions**

3) **A deterministic method** that others can re-run and get the same file list and hashes

4) **Invariants the corpus must satisfy** (examples)

5) **Negative / adversarial tests** (see below)  
   Not just “I found 10,000 files,” but “I attempted 10 common hiding tricks and found none.”

6) **Provenance and freshness**  
   When was the scan run, on which machine path, and under what tool versions.

Without (2)–(3), a completeness claim is mostly vibes.

## How we (or a reviewer) can be tricked: the threat model

### 1) **Universe smuggling (the biggest one)**  

A report looks exhaustive because it hits “tons of `.md`,” but it never searched `~/Documents`, an external drive, a private org repo, a cloud doc store, or a different branch.  
_Symptom:_ huge counts + warm feelings; _still not closed-world_.

### 2) **Name/path hiding**  

Writing stored as:

- `README` with no extension  
- `*.mdx` / `*.org` / `*.typ` / `*.ipynb` / PDF-only  
- content embedded in `*.html`, `*.json`, or SQL dumps  
_Symptom:_ glob undercounts.

### 3) **Content hiding inside non-text artifacts**  

PDF/DOCX, images, slide decks, email exports, Notion zips, chat exports.

### 4) **Shadow duplicates masquerading as “done”**  

The same “canonical” text exists in 3 places (repo + archive + distribution copy). A naive inventory inflates “done,” while a canonical one might still be missing a single authoritative locus.  
_Symptom:_ high file counts + unclear authority.

### 5) **Tool truncation / search limits (silent)**  

If any step caps results or truncates output, you can _believe_ you saw the full set while missing tail files.

### 6) **“Implementation theater”**  

You see pipelines and contracts and assume runtime guarantees exist, but some paths are scaffolds/TODO, or the evaluation gates are not actually enforced in CI.  
_Symptom:_ impressive architecture docs, weak enforcement.

### 7) **Hand-wavy “audit complete” without artifacts**  

A narrative audit (“all repairs complete…”) is not a proof unless backed by a manifest, tests, and reproducible diffs.

### 8) **Symlinks, ignored dirs, and permission silos**  

A scan that skips permission errors will silently undercount.
